# 구글 스프레드시트 → Firebase 이전 가이드

아래 순서대로만 따라하시면 됩니다. 어려운 용어는 없으니 천천히 클릭만 하시면 돼요.

---

## 1단계. Firebase 프로젝트 만들기

1. https://console.firebase.google.com 접속 (구글 계정으로 로그인)
2. **"프로젝트 추가"** 클릭
3. 프로젝트 이름 입력 (예: `우리동네약국`) → 계속
4. Google 애널리틱스는 **끄기(사용 안 함)** 선택해도 무방 → 프로젝트 만들기
5. 몇 초 기다리면 완료

## 2단계. Firestore 데이터베이스 만들기

1. 왼쪽 메뉴에서 **빌드(Build) → Firestore Database** 클릭
2. **"데이터베이스 만들기"** 클릭
3. 위치는 **asia-northeast3 (서울)** 선택 (한국에서 가장 빠름)
4. 보안 규칙은 일단 **"테스트 모드로 시작"** 선택 → 사용 설정

> 테스트 모드는 30일 후 자동으로 잠기니, 아래 4단계에서 규칙을 직접 넣어줄 거예요.

## 3단계. 웹 앱 등록하고 설정 값(config) 복사하기

1. 프로젝트 개요 화면(홈)에서 **`</>`(웹) 아이콘** 클릭
2. 앱 닉네임 아무거나 입력 (예: `약국앱`) → **Firebase 호스팅은 체크 안 함** → 앱 등록
3. 화면에 나오는 코드 중 아래처럼 생긴 부분을 찾으세요:

```js
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "xxx.firebaseapp.com",
  projectId: "xxx",
  storageBucket: "xxx.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef"
};
```

4. 이 부분을 통째로 복사해서, 제가 드린 **`firebase-config.js`** 파일을 열고 `여기에_붙여넣기` 부분들을 실제 값으로 교체해주세요. (메모장으로 열어서 수정하시면 됩니다)

## 4단계. 보안 규칙 설정하기

1. Firestore Database 화면 상단의 **"규칙(Rules)"** 탭 클릭
2. 아래 내용을 통째로 붙여넣고 **"게시"** 클릭

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

> ⚠️ 참고: 이 규칙은 "누구나 읽고 쓸 수 있음" 규칙이에요. 지금 앱이 로그인 시스템을 자체적으로 구현한 방식(닉네임+비밀번호를 그냥 문서로 저장)이라 원래도 아주 강한 보안은 아니었어요. 지금은 속도 개선이 목표이니 이 규칙으로 시작하시고, 나중에 보안을 더 강화하고 싶으시면 말씀해주세요.

## 5단계. 파일 교체하기

기존에 쓰시던 폴더(웹호스팅 하시는 곳, 예: Netlify, 카페24, 등)에서 아래 파일들을 **전부 새 파일로 교체/추가**해주세요.

- `index.html` → 교체
- `script.js` → 교체
- `board.js` → 교체
- `style.css` → 교체 (내용은 기존과 동일, 그대로 두셔도 됩니다)
- `firebase-config.js` → **새로 추가** (3단계에서 값 채운 파일)
- `firebase-db.js` → **새로 추가**

## 6단계. 기존 데이터 옮기기 (딱 1번만!)

1. 위 파일들과 같은 폴더에 `migrate.html` 파일도 함께 올려주세요.
2. 브라우저에서 `내주소.com/migrate.html` 로 접속
3. **"지금 데이터 옮기기"** 버튼 클릭
4. 화면에 "🎉 모든 데이터 이전이 끝났습니다!" 문구가 뜨면 성공
5. 확인 후 `migrate.html` 파일은 삭제해주세요 (다시 누르면 중복 등록될 수 있어요)

## 7단계. 최종 확인

1. `index.html`(원래 앱 주소)로 접속해서 로그인, 상품 담기, 건의게시판까지 평소처럼 잘 되는지 확인
2. 잘 되면 끝! 이제 훨씬 빨라진 걸 느끼실 거예요.
3. 기존 구글 스프레드시트/Apps Script는 당장 지우지 마시고, 며칠 정도는 백업용으로 남겨두시는 걸 추천드려요.
